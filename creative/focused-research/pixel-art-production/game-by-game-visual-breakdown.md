## 3. Dead Cells (2018)

### Art Style
Vibrant, high-contrast pixel art created through innovative 3D-to-2D pipeline

### Character Sprites
- **Size**: Medium-high detail
- **Detail Level**: Rich combat variety
- **Animation Quality**: Buttery smooth, combat-focused
- **Pipeline**: 3D models converted to pixel art

### Animation Techniques
- **3D-to-2D Workflow**:
  1. Draw 2D pixel art model sheet
  2. Create 3D character and skeleton in 3DS Max
  3. Animate with keyframes
  4. Add interpolated frames between keyframes
  5. Custom program renders at small size without antialiasing for pixelated look
  6. Export with normal maps for volume rendering
- **Attack animations**: Posture-based, VFX adds movement/strength/power
- **Easy retakes**: Key advantage of 3D pipeline for iterative combat design
- **Frame export**: Includes normal maps for toon shader effects

### Environmental Art
- Multi-layered backgrounds
- Procedurally arranged elements
- Distinct biome identities

### Color Palette
- **Color Count**: Vibrant, high-contrast (32+ colors)
- **Strategy**: Bold, saturated colors for visual clarity
- **Mood**: Intense, energetic, dark fantasy

### Lighting & Effects
- Hit effects with impact
- Volume rendering using basic toon shader
- Particle systems for combat
- Screen effects for feedback

### UI/HUD Design
- Clear health/resource indicators
- Minimal screen clutter for combat focus

### Technical Approach
- **Pipeline**: 3DS Max → Custom Renderer → Game Engine
- **Key Benefit**: Retakes are easy compared to hand-drawn
- **Philosophy**: Satisfying combat is foundation, workflow enables iteration
- Most animations are 3D pre-rendered, some simpler ones hand-made

### What We Can Adapt
- **CRITICAL FOR RHYTHM RPG**: 3D-to-2D pipeline enables rapid iteration on rhythm attack timings
- Create base animations in 3D, adjust timing until it feels perfect with music
- Use normal maps for dynamic lighting during musical crescendos
- Design posture-based attack animations that align with beat patterns
- Export variations quickly for different rhythm patterns
- Build VFX library that adds impact to rhythm hits

## 9. Shovel Knight (2014)

### Art Style
"Rose-tinted" NES - how an 8-bit game would feel if made today

### Character Sprites
- **Size**: 400×240 resolution (NES vertical, modern horizontal)
- **Detail Level**: Generally 4 colors per sprite (5 max for diversity)
- **Animation Quality**: Smooth with modern design lessons
- **Readability**: Larger color blobs replace dither

### Animation Techniques
- Eliminated sprite flickering (detrimental to gameplay)
- Avoided screen clutter
- Limited particle effects (by NES-inspired design)
- Smooth character movement

### Environmental Art
- 16:9 resolution provides breathing room
- Additional horizontal space for level design
- NES-style tile sets with modern flourishes

### Color Palette
- **Color Count**: NES restrictions (4 colors/sprite)
- **Strategy**: Hand-picked saturation boost, not mathematically regular
- **Added colors**: "Cheater colors" like light brown for skin diversity
- **Mood**: Nostalgic but vibrant

### Lighting & Effects
- Traditional NES-style (limited)
- Removed dither patterns for modern display readability

### UI/HUD Design
- Classic NES-style HUD
- Clear, readable information

### Technical Approach
- Virtual resolution of 400×240 (each pixel is 4.5×4.5 at 1080p)
- NES output was 256×240, same vertical
- "Rose-tinted view" philosophy
- Gameplay benefits from modern design
- Tech receives subtle but substantial upgrades

### What We Can Adapt
- Rose-tinted approach: Make retro feel how we remember, not how it was
- Eliminate negative retro artifacts (flickering, clutter)
- Keep positive retro constraints (limited colors, readable designs)
- Boost saturation for modern vibrant feel
- Add "cheater" elements where they improve experience
- Design levels with breathing room (16:9 consideration)
- Use larger color blobs for readability
