# Visual Direction & Asset Catalog: Complete Design Guide

---

## PART 1: VISUAL DIRECTION PHILOSOPHY

### Aesthetic Vision

**Core Concept:** "Warm, Rhythmic, Expressive Pixel Art"

Our game combines the charm of 90s pixel art with modern lighting/effects, creating an inviting world where every animation feels purposeful and musical. The visual language should communicate rhythm visually - shapes flow with music, colors pulse with beat, and movement feels choreographed.

### Color & Mood

**Primary Aesthetic:** Warm, inviting, with pops of vibrant color
**Inspiration Sources:**
- Eastward (warmth, detailed environments)
- Sea of Stars (modern pixel art with effects)
- Hyper Light Drifter (moody atmosphere)
- Celeste (expressive feedback animations)

### Key Principles

1. **Animation = Communication** - Every animation conveys meaning rhythmically
2. **Color Tells Story** - Color choice indicates mood, danger, musicality
3. **Clarity First** - Even in visual complexity, gameplay clarity maintained
4. **Expressive Over Detailed** - Animation quality > sprite frame count
5. **Integrated Music** - Visuals pulse/move with music naturally
6. **Accessibility** - Color-blind friendly palettes, clear visual feedback

## PART 7: ASEPRITE WORKFLOW GUIDE

### Canvas Sizes & Standards

**Sprite Sizes:**
- **Tile Size:** 16x16 pixels (basic tiles)
- **Large Tile:** 32x32 pixels (detailed tiles)
- **Character Base:** 32x32 pixels (fits standard tile)
- **Large Character:** 64x64 pixels (bosses, important NPCs)
- **UI Elements:** 16x16 to 64x64 depending on purpose

### Layer Organization Standards

**Character Sprite Structure:**
```
Folder: Character_Name
├── Body (base character)
├── Clothes (optional outfit layer)
├── Accessories (equipment visible)
├── Expressions (face variations)
├── Effects (auras, particles)
└── Backup (original reference)
```

**Environment Tile Structure:**
```
Folder: Tileset_Name
├── Base Layer (ground/walls)
├── Decoration (details, flora)
├── Objects (interactive items)
├── Hazards (dangerous elements)
└── Effects (animations, shadows)
```

### Animation Frame Standards

**Movement Animations:**
- **Idle:** 4-6 frames, 100-150ms per frame
- **Walk:** 6-8 frames, 100ms per frame (rhythmic)
- **Run:** 8 frames, 75ms per frame (faster rhythm)
- **Attack:** 6-10 frames, varies by weapon
- **Dodge/Dash:** 4-6 frames, 50ms per frame (quick)
- **Special Move:** 10-15 frames, variable timing

**Dialogue/Expression:**
- **Talking:** 2-4 frames, cycles while dialogue plays
- **Surprised:** 1 frame (instant expression)
- **Happy:** 3-4 frames, 150ms per frame
- **Sad:** 3-4 frames, 200ms per frame (slow)
- **Angry:** 3-4 frames, 100ms per frame (sharp)

### Color Management

**Palette Best Practices:**
- Create indexed color palette for each game
- Use consistent color indices across sprites
- Leave transparent color (usually color 0)
- Document color meanings in spreadsheet

**Export Settings:**
- Format: PNG with transparency
- Bit Depth: 8-bit indexed (16x16 tiles)
- Bit Depth: 16-bit RGBA (larger sprites with effects)
- Filter: Nearest neighbor (preserve pixel art)

### Animation Export for Godot

**Sprite Sheet Organization:**
- Rows: Different action states (idle, walk, run, attack)
- Columns: Animation frames
- Export as single sprite sheet PNG
- Document frame sizes and frame counts

**Metadata for Godot:**
- Create JSON file describing:
  - Frame size (pixels)
  - Frames per action
  - Frames per second (animation speed)
  - Collision/hitbox data if needed

### Batch Processing Workflow

**Efficient Production Pipeline:**
1. Create reference art in fullsize
2. Scale down to final size
3. Optimize colors (256 color palette max)
4. Animate in Aseprite
5. Export sprite sheets
6. Generate JSON metadata
7. Import into Godot

### Keyboard Shortcuts (Aseprite)

**Essential Shortcuts:**
- `N` - New frame
- `K` - Delete frame
- `,` / `.` - Previous/Next frame
- `Space + Mouse` - Pan canvas
- `Shift + Mouse Wheel` - Zoom
- `Ctrl + Alt + I` - Invert colors (useful for quick previews)

### Design Tips

**Pixel Art Best Practices:**
- Use 1-pixel outlines sparingly
- Anti-aliasing rarely needed (keep it sharp)
- Consistent pixel density across sprites
- Use limited palette per character (8-16 colors)
- Animate in 2s (every other frame for smoothness)

**Performance Optimization:**
- Limit animation frames (4-8 for most movements)
- Use sprite atlasing (combine related sprites)
- Compress PNG files post-export
- Consider animation caching in Godot

## PART 8: ANIMATION SPECIFICATIONS

### Character Animations by State

**Idle Animation (Neutral, comfortable):**
- 4-6 frames
- Subtle breathing/swaying motion
- 100-150ms per frame
- Loops smoothly
- No dramatic movement

**Walk Animation (Movement at normal speed):**
- 6-8 frames (alternating legs)
- Maintain rhythm with music tempo
- 100ms per frame baseline
- Arm swing opposite legs
- Smooth weight shifting

**Run Animation (Fast movement):**
- 8 frames (rapid legs)
- More exaggerated limb movement
- 75ms per frame (faster)
- Energy transfer visible
- Dynamic weight shift

**Attack Animations (Varies by weapon):**
- Sword: 6-8 frames, 50ms per frame (quick, sharp)
- Hammer: 8-10 frames, 75ms per frame (windup/impact)
- Bow: 10 frames, includes draw + release
- Magic: 12-15 frames, includes buildup + release
- All emphasize beat synchronization visually

**Special Move Animation (Ultimate abilities):**
- 15-20 frames minimum
- Impressive visual flourish
- Dramatic timing matched to music
- Clear beginning, middle, end
- Followed by brief recovery pose

**Dodge/Dash Animation (Evasion):**
- 4-6 frames, 50ms per frame
- Direction-specific (8 directions ideally)
- Quick, reactive feeling
- Clear visual indication of invincibility window
- Smooth return to idle

### Environmental Animations

**Platform Animations:**
- Movement: 2-4 frames based on rhythm
- Pulsing: Growing/shrinking in sync with music
- Appearing/Disappearing: Smooth fade in/out
- Hazard activation: Clear visual warning before activation

**Weather Effects:**
- Rain: Falling animation, 2 frames minimum
- Snow: Floating animation, 4-6 frames
- Wind: Swaying/pushing animation, 3-4 frames
- Lightning: Flash effect, instant or 1-2 frames

**Combat Effect Animations:**
- Hit Impact: Flash/shake, 2-4 frames
- Particle Burst: Explosion-like effect, 8-12 frames
- Buff Aura: Glowing effect, 4-6 frames
- Damage Number: Floating text, moves + fades over time

## IMPLEMENTATION NOTES

### Asset Production Timeline
- Characters: 2-3 weeks (30 sprites, 6-8 animations each)
- Environments: 3-4 weeks (5 biomes, 20-30 tiles each)
- UI: 1-2 weeks (50+ unique elements)
- VFX: 2-3 weeks (50+ effect variations)
- Total: 8-12 weeks of art production

### Godot 4.5+ Integration
- Use **AtlasTexture** for sprite sheets
- Implement **AnimatedSprite2D** with frame-by-frame control
- Connect animations to **AudioStreamPlayer** for sync
- Use **Shaders** for color effects and rhythmic glow
- Optimize with **CanvasItem** culling

### Accessibility Considerations
- Color-blind friendly palettes primary and secondary
- High contrast mode available
- Clear UI without relying on color alone
- Readable fonts with adequate spacing
- Clear visual feedback for all important actions
